# FLUX.1-schnell INT8 OpenVINO モデル特徴

103枚の生成テスト結果に基づく。

## 1. スタイル制御力（題材別）

| 題材 | リアル↔アニメ | ドット絵 | 水彩/油絵/フラット等 | 備考 |
|------|:---:|:---:|:---:|------|
| **人物（クローズアップ）** | × | × | ○ | リアル/アニメ/ドットの3軸はほぼ区別不能。アニメ調に収束 |
| **人物（群衆・遠景）** | △ | △ | ○ | 人が小さいと差が出やすい |
| **人物（ファンタジーキャラ）** | △ | △ | ○ | エルフ等はリアルがやや出る。ドットは半端 |
| **動物（猫等）** | ○ | ◎ | ◎ | 全スタイル対応。テスト中最も優秀 |
| **車・乗り物** | ◎ | ◎ | ○ | リアル/アニメ/ドット明確に分離 |
| **建物・風景** | ◎ | ◎ | ○ | 3スタイルきれいに分離 |
| **アイテム（剣・宝箱等）** | ◎ | ◎ | ○ | 最もスタイル制御が効く。ゲームアセットに最適 |
| **食べ物** | × | ○ | ○ | リアルとアニメが近い。ドットは効く |
| **メカ・ロボット** | ○ | ◎ | ○ | ドットが特に効く |
| **ドラゴン等モンスター** | ○ | ◎ | ○ | ドットが特に効く |

### まとめ
- **人物のクローズアップだけがスタイル制御困難**（リアル/アニメ/ドットの3軸）
- 水彩・フラット・ちび・浮世絵・サイバーパンク等の**具体的な画風指定は人物でも効く**
- アイテム・乗り物・建物は全スタイル自在

## 3. 人物のスタイル制御テクニック

人物はデフォルトでアニメ調に収束するが、以下で突破可能:

### リアル化
- `Canon EOS R5, 85mm f1.4, shallow depth of field, professional headshot` — カメラ機種名が効く
- `Unreal Engine 5, subsurface scattering, ray tracing` — 3DCG文脈で突破
- 「portrait photograph」より「Canon EOS R5」のほうが強い

### ドット化（迂回策）
- `Minecraft character, voxel human, blocky 3D` — ボクセルとして指定
- `RPG game sprite sheet, 32x32 pixel character, top-down view` — スプライトシートとして指定
- 単なる「pixel art」では人物に効かない

### 確実に効くスタイル変更
- 水彩、油絵、フラットデザイン、鉛筆スケッチ、浮世絵、ちび、サイバーパンク
- これらは人物でも明確にスタイルが変わる

## 4. Japanese aesthetic prefix の効果

| 効果 | 詳細 |
|------|------|
| **背景モチーフ** | 和風建築、鳥居、桜、月が出現しやすくなる |
| **装飾** | 丼の模様が和柄になる、衣服に和風要素が追加される |
| **色味** | 紫・朱赤・金の和風カラーパレットに寄る |
| **漢字** | 画面に意味不明な漢字風文字が出現する（副作用） |
| **スタイル干渉** | 小さい。リアル/アニメ/ドットの制御を邪魔しない |

**結論**: 和風テイストを出したいなら有効。ただし元々モデルがアジア寄りなので、効果は「微妙に和風が増す」程度。邪魔にはならない。

## 5. サイズと生成時間

| サイズ | 生成時間 | 品質 | 備考 |
|--------|---------|------|------|
| 256×256 | ~9秒 | 低（細部が潰れる） | サムネイル・アイコン向き |
| 512×512 | ~14秒 | 標準 | **推奨デフォルト** |
| 768×768 | ~55秒 | 高 | 細部が増えるが3.5倍遅い |
| 1024×1024 | ~120秒 | 最高 | 8.5倍遅い。ここぞという時のみ |

- 768以上ではスタイルがやや変わることがある（フラット→描き込み寄りになる等）
- コスパは512が最良

## 6. ステップ数の影響

| ステップ | 生成時間(512) | 効果 |
|---------|-------------|------|
| 4 | ~14秒 | 標準。schnellの設計値 |
| 8 | ~75秒 | 描き込みUP。リアル系は改善するが、ドット絵は逆にドット感が薄れる |

**結論**: 4ステップが基本。リアル写真風を出したいときだけ8ステップを検討。ドット絵には逆効果。

## 7. テキスト描画

- 漢字・日本語: 意味不明な文字が生成される（読めない）
- 英語: 一部読めるが不安定
- **テキストが必要な画像は後加工推奨**

## 8. 推奨用途

| 用途 | 適性 | 理由 |
|------|:---:|------|
| ゲームアイテム・武器・宝箱 | ◎ | スタイル制御◎、アセットとして即使える |
| ゲーム背景・風景 | ◎ | スタイル自在、高品質 |
| アニメ風キャラクター | ◎ | デフォルトでこれ。得意中の得意 |
| ちびキャラ・マスコット | ◎ | ちびスタイルが効く |
| UIアイコン（フラット） | ○ | フラットデザイン指定が効く |
| 動物イラスト | ◎ | 全スタイル対応 |
| リアル人物写真 | △ | カメラ機種名指定で可能だが不安定 |
| ドット絵キャラ | △ | Minecraft/スプライトシート指定で迂回可能 |
| テキスト入り画像 | × | 読める文字は生成できない |
| ロゴ・タイポグラフィ | × | テキスト描画不可 |
